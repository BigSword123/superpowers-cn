# 规格合规审查员提示模板

在派发规格合规审查子智能体时使用此模板。

**目的：** 验证实现者构建了被请求的东西（不多，不少）

```
Task 工具（通用型）：
  description: "审查任务 N 的规格合规性"
  prompt: |
    你正在审查一个实现是否与其规格匹配。

    ## 被请求的内容

    [任务需求的完整文本]

    ## 实现者声称他们构建了什么

    [来自实现者的报告]

    ## 关键：不要信任报告

    实现者完成得异常快。他们的报告可能不完整、
    不准确或过于乐观。你必须独立验证所有内容。

    **不要做：**
    - 相信他们对自己实现了什么的说辞
    - 信任他们关于完整性的声明
    - 接受他们对需求的解释

    **要做：**
    - 阅读他们实际写的代码
    - 将实际实现逐行与需求对比
    - 检查他们声称实现但可能缺失的部分
    - 寻找他们没有提到的额外功能

    ## 你的工作

    阅读实现代码并验证：

    **缺失的需求：**
    - 他们是否实现了所有被请求的内容？
    - 是否有他们跳过或遗漏的需求？
    - 他们是否声称某些东西可以工作但实际没有实现？

    **额外/不需要的工作：**
    - 他们是否构建了没有被请求的东西？
    - 他们是否过度工程或添加了不必要的功能？
    - 他们是否添加了规格中没有的"锦上添花"功能？

    **误解：**
    - 他们是否以不同于预期的方式解释需求？
    - 他们是否解决了错误的问题？
    - 他们是否实现了正确的功能但用了错误的方式？

    **通过阅读代码来验证，而不是信任报告。**

    报告：
    - ✅ 符合规格（如果代码检查后一切匹配）
    - ❌ 发现问题：[具体列出缺失或额外的东西，包含文件:行号引用]
```
