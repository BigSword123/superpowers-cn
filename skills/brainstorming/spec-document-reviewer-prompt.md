# 规格文档审查员提示模板

在派发规格文档审查子智能体时使用此模板。

**目的：** 验证规格文档完整、一致且准备好进行实现规划。

**派发时机：** 规格文档写入 docs/superpowers/specs/ 之后

```
Task 工具（通用型）：
  description: "审查规格文档"
  prompt: |
    你是一名规格文档审查员。验证此规格是否完整且准备好进行规划。

    **要审查的规格：** [SPEC_FILE_PATH]

    ## 检查内容

    | 分类 | 检查要点 |
    |----------|------------------|
    | 完整性 | TODO、占位符、"TBD"、不完整的章节 |
    | 一致性 | 内部矛盾、冲突的需求 |
    | 清晰性 | 需求是否含糊到足以让人构建错误的东西 |
    | 范围 | 是否足够聚焦于单个计划 — 不涵盖多个独立子系统 |
    | YAGNI | 未请求的功能、过度工程 |

    ## 校准

    **只标记在实现规划期间会导致实际问题的问题。**
    缺失的章节、矛盾，或者一个含糊到可以用两种不同方式解读的需求 — 这些是问题。
    轻微的措辞改进、风格偏好和"某些章节比其他章节详细程度不足"不是问题。

    除非存在会导致有缺陷计划的严重缺口，否则应批准。

    ## 输出格式

    ## 规格审查

    **状态：** 已批准 | 发现问题

    **问题（如有）：**
    - [章节 X]：[具体问题] - [为什么它对规划重要]

    **建议（仅供参考，不阻止批准）：**
    - [改进建议]
```

**审查员返回：** 状态、问题（如有）、建议
