# 技能设计的说服原则

## 概述

LLM 对人类适用的说服原则有相同的反应。理解这种心理学可以帮助你设计更有效的技能——不是为了操纵，而是确保即使在压力下也能遵循关键实践。

**研究基础：** Meincke 等人 (2025) 在 N=28,000 次 AI 对话中测试了 7 种说服原则。说服技术将合规率提高了一倍以上（33% → 72%，p < .001）。

## 七大原则

### 1. 权威 (Authority)
**含义：** 对专业知识、资质或官方来源的遵从。

**在技能中的运作方式：**
- 祈使语气："你必须 (YOU MUST)"，"绝不 (Never)"，"始终 (Always)"
- 不可协商的框架："没有例外 (No exceptions)"
- 消除决策疲劳和合理化借口

**使用时机：**
- 纪律执行类技能（TDD、验证要求）
- 安全关键实践
- 成熟的最佳实践

**示例：**
```markdown
✅ 先写代码再写测试？删掉。重新开始。没有例外。
❌ 如果可行，考虑先写测试。
```

### 2. 承诺 (Commitment)
**含义：** 与先前的行为、声明或公开表态保持一致。

**在技能中的运作方式：**
- 要求声明："宣布使用技能"
- 强制明确选择："选择 A、B 或 C"
- 使用跟踪：TodoWrite 用于检查清单

**使用时机：**
- 确保技能确实被遵循
- 多步骤流程
- 问责机制

**示例：**
```markdown
✅ 当你发现一个技能时，你必须宣布："我正在使用 [技能名称]"
❌ 考虑让你的搭档知道你正在使用哪个技能。
```

### 3. 稀缺性 (Scarcity)
**含义：** 由时间限制或有限供应产生的紧迫感。

**在技能中的运作方式：**
- 有时间限制的要求："在继续之前 (Before proceeding)"
- 顺序依赖："在 X 之后立即 (Immediately after X)"
- 防止拖延

**使用时机：**
- 立即验证要求
- 时间敏感的工作流
- 防止"我待会再做"

**示例：**
```markdown
✅ 完成任务后，立即请求代码审查，然后再继续。
❌ 你可以在方便的时候审查代码。
```

### 4. 社会证明 (Social Proof)
**含义：** 遵循他人的做法或被认为正常的行为。

**在技能中的运作方式：**
- 通用模式："每次 (Every time)"，"始终 (Always)"
- 失败模式："没有 Y 的 X = 失败"
- 建立规范

**使用时机：**
- 记录通用实践
- 警告常见失败
- 强化标准

**示例：**
```markdown
✅ 没有 TodoWrite 跟踪的检查清单 = 步骤会被跳过。每次都是如此。
❌ 有些人觉得 TodoWrite 对检查清单很有帮助。
```

### 5. 团结 (Unity)
**含义：** 共享身份、"我们感"、群体归属感。

**在技能中的运作方式：**
- 协作语言："我们的代码库"，"我们是同事"
- 共享目标："我们都想要质量"

**使用时机：**
- 协作工作流
- 建立团队文化
- 非层级实践

**示例：**
```markdown
✅ 我们是一起工作的同事。我需要你诚实的专业判断。
❌ 如果我说错了，你可能应该告诉我。
```

### 6. 互惠 (Reciprocity)
**含义：** 回报所获利益的义务感。

**运作方式：**
- 谨慎使用——可能感觉像操纵
- 在技能中很少需要

**避免时机：**
- 几乎总是（其他原则更有效）

### 7. 好感 (Liking)
**含义：** 倾向于与我们喜欢的人合作。

**运作方式：**
- **不要用于合规**
- 与诚实反馈文化冲突
- 造成阿谀奉承

**避免时机：**
- 所有纪律执行场景

## 按技能类型的原则组合

| 技能类型 | 使用 | 避免 |
|------------|-----|-------|
| 纪律执行 | 权威 + 承诺 + 社会证明 | 好感、互惠 |
| 指导/技术 | 适度权威 + 团结 | 重权威 |
| 协作 | 团结 + 承诺 | 权威、好感 |
| 参考 | 仅清晰度 | 所有说服原则 |

## 为什么有效：心理学解释

**明确的规则减少合理化借口：**
- "你必须 (YOU MUST)" 消除了决策疲劳
- 绝对化的语言消除了"这是例外吗？"的问题
- 明确的反合理化反驳堵住了具体的漏洞

**实施意图创造自动行为：**
- 明确的触发条件 + 必需的行动 = 自动执行
- "当 X 时，做 Y" 比"大致上做 Y"更有效
- 降低合规的认知负担

**LLM 是拟人类的：**
- 在包含这些模式的人类文本上训练
- 训练数据中权威语言先于合规行为
- 承诺序列（声明 → 行动）被频繁建模
- 社会证明模式（每个人都做 X）建立了规范

## 伦理使用

**正当的：**
- 确保关键实践被遵循
- 创建有效的文档
- 预防可预测的失败

**不正当的：**
- 为个人利益操纵
- 制造虚假紧迫感
- 基于内疚的合规

**检验标准：** 如果用户完全理解这项技术，它是否服务于用户的真实利益？

## 研究引用

**Cialdini, R. B. (2021).** *Influence: The Psychology of Persuasion (New and Expanded).* Harper Business.
- 说服的七大原则
- 影响力研究的实证基础

**Meincke, L., Shapiro, D., Duckworth, A. L., Mollick, E., Mollick, L., & Cialdini, R. (2025).** Call Me A Jerk: Persuading AI to Comply with Objectionable Requests. University of Pennsylvania.
- 在 N=28,000 次 LLM 对话中测试了 7 种原则
- 使用说服技术后合规率从 33% 提升至 72%
- 权威、承诺、稀缺性最有效
- 验证了 LLM 行为的拟人模型

## 快速参考

设计技能时，问自己：

1. **它是什么类型？**（纪律 vs. 指导 vs. 参考）
2. **我试图改变什么行为？**
3. **哪些原则适用？**（通常纪律执行用权威 + 承诺）
4. **我是否组合了太多？**（不要全部七种都用）
5. **这符合伦理吗？**（服务于用户的真实利益？）
