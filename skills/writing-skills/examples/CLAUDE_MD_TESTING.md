测试不同的文档变体，以找到什么真正让智能体在压力下发现并使用技能。

## 测试场景

```

```

## 要测试的文档变体

### NULL（基准 — 没有技能文档）
在 CLAUDE.md 中完全没有提到技能。

### 变体 A：柔和建议
```markdown
## 技能库

你可以访问 `~/.claude/skills/` 中的技能。考虑
在开始任务之前检查相关技能。
```

```markdown

在开始任何任务之前，检查 `~/.claude/skills/` 中

浏览：`ls ~/.claude/skills/`
搜索：`grep -r "keyword" ~/.claude/skills/`
```

## 测试协议

对于每个变体：

1. **首先运行 NULL 基准**（没有技能文档）
   - 记录智能体选择哪个选项
   - 捕获确切的合理化

2. **运行变体**使用相同场景
   - 智能体检查技能了吗？
   - 如果找到，智能体使用技能了吗？
   - 如果违反，捕获合理化

3. **压力测试** — 添加时间/沉没成本/权威
   - 智能体在压力下仍然检查吗？
   - 记录合规在何时崩溃

4. **元测试** — 问智能体如何改进文档
   - "你有文档但没检查。为什么？"
   - "文档怎样能更清晰？"

## 成功标准

**变体成功如果：**
- 智能体在没有提示的情况下检查技能
- 智能体在行动之前完整阅读技能
- 智能体在压力下遵循技能指导
- 智能体无法合理化绕过合规

**变体失败如果：**
- 智能体即使没有压力也跳过检查
- 智能体"适应概念"而不阅读
- 智能体在压力下合理化绕过
- 智能体将技能视为参考而非要求

## 预期结果

**NULL：** 智能体选择最快路径，无技能意识

**变体 A：** 智能体可能在无压力时检查，在压力下跳过

**变体 B：** 智能体有时检查，容易合理化绕过

**变体 C：** 强合规但可能感觉过于僵硬

**变体 D：** 平衡，但更长 — 智能体会内化它吗？
