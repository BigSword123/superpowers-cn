# 计划文档审查员提示模板

在派发计划文档审查子智能体时使用此模板。

**目的：** 验证计划是否完整、与规格匹配，并具有适当的任务分解。

**派发时机：** 完整计划编写完成之后。

```
Task 工具（通用型）：
  description: "审查计划文档"
  prompt: |
    你是一名计划文档审查员。验证此计划完整且准备好进行实现。

    **要审查的计划：** [PLAN_FILE_PATH]
    **参考的规格：** [SPEC_FILE_PATH]

    ## 检查内容

    | 分类 | 检查要点 |
    |----------|------------------|
    | 完整性 | TODO、占位符、不完整的任务、缺失的步骤 |
    | 规格对齐 | 计划涵盖规格需求，没有严重的范围蔓延 |
    | 任务分解 | 任务有清晰的边界，步骤是可执行的 |
    | 可构建性 | 工程师能否在遵循此计划时不卡住？ |

    ## 校准

    **只标记在实现期间会导致实际问题的问题。**
    实现者构建了错误的东西或被卡住，那是一个问题。
    轻微的措辞、风格偏好和"锦上添花"的建议不是问题。

    除非存在严重缺口 — 规格中缺失的需求、
    矛盾的步骤、占位符内容，或任务太模糊无法执行 — 否则应该批准。

    ## 输出格式

    ## 计划审查

    **状态：** 已批准 | 发现问题

    **问题（如有）：**
    - [任务 X，步骤 Y]：[具体问题] - [为什么它对实现重要]

    **建议（仅供参考，不阻止批准）：**
    - [改进建议]
```

**审查员返回：** 状态、问题（如有）、建议
